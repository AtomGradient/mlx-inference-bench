Key Findings核心发现

⚡

Memory Bandwidth is the Wall内存带宽是天花板

MLX achieves 85% of M2 Ultra's 800 GB/s bandwidth during decode. Batch forward passes cannot approach theoretical speedup because each query token linearly increases KV cache bandwidth demand. MLX decode 已达 M2 Ultra 800 GB/s 带宽的 85%。Batch 前向传播无法接近理论加速比，因为每个额外 query token 会线性增加 KV cache 带宽需求。

❌

Speculative Decoding Doesn't Help投机解码无法加速

N-gram speculative decoding yields 0.85-1.00x on both pure Transformer and hybrid architectures. The pure Transformer (1.79x batch speedup) is actually worse than the hybrid (2.57x). N-gram 投机解码在纯 Transformer 和混合架构上均为 0.85-1.00x。纯 Transformer 的 batch 加速比 (1.79x) 反而低于混合架构 (2.57x)。

🔎

Prefill Bottleneck is GEMM, Not AttentionPrefill 瓶颈是 GEMM 不是 Attention

Quantized matrix multiplication in linear projections accounts for 57.6% of prefill time. SDPA attention is only 6.7%. Optimizing attention kernels alone cannot close the gap with llama.cpp. 量化矩阵乘法占 prefill 时间的 57.6%，SDPA attention 仅占 6.7%。单独优化 attention kernel 无法弥合与 llama.cpp 的差距。

✅

MLX is More Mature Than ExpectedMLX 比预期更成熟

AOT shader compilation, single-dispatch attention, and SIMD matrix multiply are already implemented. Five "missing features" from the prior proposal actually exist. AOT 预编译、单次 dispatch attention、SIMD 矩阵乘法都已实现。原方案中 5 个"缺失特性"实际已存在。

Corrected Misconceptions误解纠正

#1 "MLX uses 256+ kernel dispatches" → MLX uses a single dispatch_threadgroups call, identical to llama.cpp. "MLX 使用 256+ 次 dispatch" → MLX 使用 1 次 dispatch_threadgroups 调用，与 llama.cpp 完全相同。
#2 "Metal shaders are JIT-compiled" → MLX defaults to AOT compilation (mlx.metallib). JIT is an optional dev mode. "Metal shader 是 JIT 编译的" → MLX 默认使用 AOT 预编译 (mlx.metallib)，JIT 是可选的开发模式。
#3 "SIMD matrix multiply is not utilized" → MLX's steel_attention and steel/gemm already use simdgroup_multiply_accumulate. "SIMD 矩阵乘法未被利用" → MLX 的 steel_attention 和 steel/gemm 已经使用 simdgroup_multiply_accumulate。
#4 "Pure Transformer batch(5) should approach 5x" → Actual speedup is only 1.79x due to memory bandwidth saturation. "纯 Transformer batch(5) 应接近 5x" → 实际加速比仅 1.79x，因为内存带宽已饱和。
#5 "Speculative decoding should yield 1.5-2x" → N-gram speculative decoding yields 0.85-1.00x (no improvement). "投机解码预计 +1.5-2x" → N-gram 投机解码实际为 0.85-1.00x（无改善）。
